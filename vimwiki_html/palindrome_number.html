Palindrome Number

题目要求 ：

  1. 判断一个给定整数是否回文。
  2. 不要用额外的空间。

题目分析 ：

  • 负整数有可能是回文吗？
  • 如果想采用把int型转化为string类型再做判断的话，注意题目要求，不要占用额外的空间。
  • 如果采用反转数字的方法，需要考虑可能 溢出 的情况。

解法分析 ：

  • 对于一个回文的整数，从前往后到中间数字，应该等于从后往前到中间数字。
  • 但是要考虑最后一位等于0的情况。不做处理的话，12210可能被认为回文。
  • 由于参数的位数可能是奇数位，也可能是偶数位。偶数位的话，最后除完x是等于n的，但是若是奇数位，我们并不care中间那一位，只需要剩下的部分相等就行。

case设计 ：

  • -12321
  • 12210
  • 12321
  • 123321 

class Solution {
public:
    bool isPalindrome(int x) {
        if (x < 0 || (x > 0 && x % 10 == 0)) return false;
        int n = 0;
        while(x > n)
        {
            n = n * 10 + x % 10;
            x = x/10;
        }
        return x == n || n/10 == x;
    }
};
